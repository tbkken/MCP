现在我有一个MCP管理的需求，需求要求如下： 
1、MCP服务查看： 展现所有的MCP服务列表，列表采用卡片的形式展示。
2、MCP服务添加，在页面上有新增按钮，点击按钮，打开MCP服务添加窗口。 
参考现在agent框架中，mcpServers配置项的参数，来新建MCP服务： 
MCP服务配置格式举例： 
"mcpServers": { 
    "user info system": { 
        "timeout": 60, 
        "type": "sse", 
        "url": "http://127.0.0.1:10002/sse/", 
        "headers": { "Authorization": "Bearer 54bfe0d6-803f-4f60-9bed-0d7709731011" } 
    } 
} 
新增MCP服务有两种输入方式： 
第一种是传统的FORM表单的方式，让用户一个个输入配置项，不常用的可选配置项默认折叠起来，
必须输入的参数为 name、type、url、headers； 
第二种是直接输入JSON字符串，前端解析成为FORM表单然后提交给后台服务。 用户填写完MCP服务后，需要有一个测试按钮，测试MCP服务是否可用。
3、MCP服务删除； 
每个MCP服务卡片上，有一个删除按钮，点击删除按钮后，弹出确认对话框，删除MCP服务。
4、MCP工具查看； 
每个MCP服务卡片上，有一个查看按钮，点击查看按钮，弹出MCP服务里面支持的所有MCP Tool。
